## 段落分けの基本方針

### 1. 期間区分（大枠）

月全体を値動きの転換点を基準に **前半・後半** または **前半・中旬・後半** の2〜3段落に分ける。

- **前半**：月初〜おおむね10日〜13日ごろまで
- **後半**：中旬以降〜月末
- **中旬**：値動きの転換が中間にある場合は3分割し、中旬を独立した段落とする

転換点の目安：
- 主要株価指数（日経平均・NYダウ・S&P500など）が明確に方向転換した日
- 月間高値・月間安値・大底を記録した日
- 重要イベント（FOMC・雇用統計・決算発表など）を契機とした相場転換

### 2. 段落内の文章分け（重要）

各段落（月前半・月後半など）の内部は、**サブ期間ごとの一文サマリー**を起点として2〜4ブロックに分類する。

**一文サマリーの役割：**
- 各サブ期間（月初・月半ば・月末にかけて、など）を **一文で要約した見出し文**
- 〇/×/△の記号を冒頭に付けて株価への影響を示す
- この一文がSKILL③での草稿文の骨格となる

**一文サマリーの下に配下の要因を階層化する：**
- 一文サマリー（第1階層）
  - 要因（第2階層）
    - 詳細・サブ要因（第3階層）

ユーザー提示の実例（2026/02 米国株）を正式フォーマットとして採用する：

```

      ×事業モデル破壊への懸念

月後半
  △後半は膠着した値動き
    〇経済指標が底堅さ、CPIが予想以下で利上げ期待が株価の下値支える
    ×△相互関税に違憲判決で関税政策に不透明感

  月末にかけて
    〇巨額AI投資継続期待で半導体関連銘柄が買われる
      〇SOX史上最高値
      ×一方で巨大IT銘柄は低調
    ×英住宅ローン会社が破綻
      ×金融株売られる
```

**フォーマットルール：**

- 段落見出し（`月前半` / `月後半` / `月中旬`）：記号なし・インデントなし
- 第1階層（2スペース）：**一文サマリー**。〇/×/△ ＋ 「月初、〜した。」「月半ばにかけて、〜した。」などの時期を明示した一文
  - 第2階層（4スペース）：要因。〇/×/△ ＋ 要因の要点
    - 第3階層（6スペース）：詳細・サブ要因。〇/×/△ ＋ 詳細
- 時期のみを示すラベル（`月末にかけて`）は記号なしでもよい
- 複数の記号が混在する場合（ポジとネガが拮抗）は `×△` や `〇△` のように並記してよい

## 転換点の特定方法

1. 分類構造化.txt の各日付エントリを時系列で確認する
2. 主要株価指数の数値データ（日経平均＝XX円、NYダウ＝XX.XXドルなど）を拾い出してトレンドを把握する
3. 上昇→下落、下落→上昇、膠着→トレンド発生、などの変化点を探す
4. 複数の転換候補がある場合は、最も値動きへの影響が大きかった点を採用する

---

## 一文サマリーの書き方

一文サマリーは**草稿文の骨格**となるため、以下の要領で作成する。

**構成要素：**
- 時期の明示：「月初、」「月半ばにかけて、」「後半は」「月末にかけて」など
- 値動きの特徴：「揉み合いで推移した」「上値が抑えられた」「膠着した値動き」「上昇した」「下落した」など
- 冒頭に〇/×/△の記号

**値動きのパターン別サマリー例：**

| 局面 | 一文サマリー例 |
|------|--------------|
| 上昇 | 〇月初、上昇基調で推移した。 |
| 上昇 | 〇月半ばにかけて、買い優勢が続いた。 |
| 下落 | ×月初、売り圧力が強まった。 |
| 下落 | ×後半にかけて、下落が加速した。 |
| 膠着 | △月初、揉み合いで推移した。 |
| 膠着 | △後半は膠着した値動きとなった。 |
| 反発 | 〇月末にかけて、反発した。 |
| 上値重い | ×△月半ばにかけて、上値が抑えられた。 |

## 段落内の構造　具体的ルール

- **1段落に一文サマリーは2〜4個**（＝その段落から生まれる草稿文の数）
- **1一文サマリーの配下に要因は1〜4項目程度**。多すぎる場合は主要なものに絞る
- 〇/×/△の記号は全ての行に付ける（時期ラベルのみの行を除く）
- 複数の影響が混在する項目は `×△` `〇△` のように並記してよい
- 数値データ（株価水準・変動幅・変動率）は要因の中に積極的に記載する

## 処理手順

```
Step 1: 東証分類構造化.txt（または米国株分類構造化.txt）を読み込む
Step 2: 各日の株価データを時系列で整理し、転換点を特定する
Step 3: 月全体を前半・後半（または前半・中旬・後半）に区分する
Step 4: 各期間内を値動きの性質ごとに2〜4の文章ブロックに分類する
Step 5: 上記フォーマットに従って月間構造化アウトラインを作成する
Step 6: 出力ファイルに書き込む
```

**Task エージェントは使用しない**。構造が定型的なため直接処理する。
